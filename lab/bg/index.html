BG.CSS

Basics

bg.css provides several classes and tools to easily build complex backgrounds. They should help your website to stand out and to give it the character you are looking for. A very basic background, such this lavender one, can be written as such : Backgrounds are rendered as a pseudo-element (::before) with a z-index of -1 and this is why you should use provided CSS Variables to manipulate it instead of using traditional CSS properties.

CSS Variables

  • --img : The url of an image to use as background (e.g. --img:url(yourimage.jpeg);).
  • --bg : A <color> to use as background.
  • --bg-position : Equivalent of CSS property background-position. Default : center.
  • --bg-size : Equivalent of CSS property background-size. Default : cover.
  • --bg-repeat : Equivalent of CSS property background-repeat. Default : no-repeat.
  • --bg-filter : Equivalent of CSS property filter. Default : none.
  • --bg-animation : Equivalent of CSS property animation, to animate the background. Default : none.
  • --bg-opacity : Opacity of the whole background. Default : 1.

JavaScript File

A small (<0.5KB) Javascript File, named bg.js is provided to fix an issue with viewport units in mobile. This file is optional and everything should work regardless. However, some margins may appears here and there.

You can include the file anywhere you want in your webpage, at any time, and everything will work by itself.

Snap background

Snap backgrounds are backgrounds that "lock" once they are in the viewport and then "unlock" once you have scroll past the content.

Snap backgrounds create impressive and visually-pleasing effects. They also help create immersive reading of the content. Because they are written in pure CSS, they are really responsive.

They are also really useful for programmers because they are height independant : They will only work if the height of the content is higher than viewport and otherwise they will act like a static background.

There are no specific CSS variables for snap backgrounds.

Split background

Split backgrounds are a serie of utilies that help you build a background made of two colors.

The minimal code for snap background is a block container with classes bg and bg-split :

The following specific variables can be used :
  • --bg1 or --bg : Your first color.
  • --bg2 : Your second color.
  • --bg-split : A dimension where the split will occur. Default : 50%.

Switch background

Inside a split background, you may want to use switch background to easily provide contrast. The width of such a block should be a defined as a percentage of its parent width and it should be placed relative to the center.
